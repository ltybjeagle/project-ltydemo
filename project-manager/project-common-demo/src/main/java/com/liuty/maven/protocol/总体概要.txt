TCP／IP（网际协议簇）：网络是在此协议簇的基础上运作的
    1、HTTP属于它内部的一个子集
    2、使用分组交换技术
    3、数据传输：首部（每一层协议相关信息） + 数据
    4、操作系统Socket库组件包：所有网络操作调用由Socket库组件包执行。
    5、路由器（路由表：静态、动态）：基于IP工作、
    6、交换机：基于以太网规则工作

TCP／IP协议模型四层体系结构：
    1、应用层：应用服务通信（HTTP、DNS、FTP）
    2、传输层：为应用层提供网路连接中两台计算机之间的数据传输（TCP、UDP）
        TCP协议：面向连接、可靠传输、性能差
        UDP协议：无连接传输、非可靠传输、性能好
    3、网络层：用来处理在网络上流动的数据包，作用就是在众多的选项内选择一条传输路线
        数据包是网络传输的最小数据单位。
        规定了通过怎样的路径到达对方计算机，并把数据包传送给对方
        IP协议：分组发送、失败不会重发（非可靠协议）
        ICMP协议：数据包发送异常，发送方接收通知
        ARP协议：根据IP查找MAC地址
    4、物理链路层：用来处理连接网络的硬件部分。
        FCS校验：硬件计算补充到数据包尾部，用于接收端校验数据是否被噪声破坏

通信数据转发程序：代理、网关、隧道
    1、代理：是一种转发功能的应用程序（服务器和客户端的中间人），不会改变请求的URI（缓存代理、透明代理）。
    2、网关：是转发其他服务器通信数据的数据器，可以使通信线路上的服务器提供非HTTP协议服务。
    3、隧道：是相隔甚远的服务器和客户端两者之间进行中转，并保持双方通信连接的应用程序。

URI（统一资源标识符）：
    1、由某个协议方案表示的资源的定位标识符。
    2、URI用字符串标识某一互联网资源。
URL（统一资源定位符）：
    1、URL表示资源的地点。
    2、URL是URI的子集。

字符编码类型：unicode(三个字节）、acsll(一个字节）
    单字节使用：同acsll码，首位为0，其他位为字符编码
    多字节使用：n字节，第一个字节前n位为1，n+1位为0，该字节剩余位为字符编码，其他字节前两位为10，
    其余6位表示字符编码
    字符编码：UTF-8是互联网上应用最广的一种unicode编码，是一种变长的编码方式，
    可以使用1--6个字节表示一个符号。
