TCP协议（传输控制协议）位于传输层，提供可靠的字节流服务（将大数据分割成已报文段为单位的数据包进行管理，
并且可以确认数据是否已经送达）。
    1、使用TCP协议前，须先建立TCP连接，形成数据传输通道。
    2、传输前，使用三次握手方式建立连接，是可靠传输。
    3、TCP协议进行通信的两个应用程序：客户端、服务端
    4、在连接中可进行大量数据的传输。
    5、传输完毕，需释放已建立的连接（四次握手），效率低。

三次握手建立连接：使用SYN、ACK标志位
四次握手断开连接：使用FIN、ACK标志位

网络包发送判定条件：
    1、根据包大小
        MTU：网络包的最大长度，一般为1500字节。
        MSS：除去头部信息，一个网络包能够容纳的TCP数据长度。
    2、根据时间，协议栈内部维护一个计数器，经过一定时间后，直接发送数据包。

数据包合包：数据包很小，可以多个数据包合成一个数据包一起发送。
数据包拆分：数据太大，一个网络包放不下
    1、网络包包含序号、长度
    2、服务端响应包包含序号，标识接收到数据。

滑动窗口：服务端响应客户端提供窗口值，告诉客户端可以发送数据长度

拥塞控制：拥塞检查在网络层、拥塞通知在传输层
    1、慢启动阶段，拥塞避免阶段。（TCP Tahoe算法）
        a、开始阶段以倍数增加发送（1、2、4、8......），当达到阀值后，再以+1的方式增加。
        b、发生阻塞，发送值调整回开始从新增量（1、2、4、8......），阀值减小（减半）。
    2、快速重传：
        a、数据是有序号的，如发送端M1、M2、M3......，接收端响应：M1收到、M2收到......。
        b、有某个数据没收到，将一直响应最后收到的数据响应（发送端可以继续发送数据）。
        c、接收端连续接收到同一个数据的回复信息，确认响应后的数据没有发送成功，则重新发送这个信息
        （重发判断：连续三次接收到同一个回复，超时未接收响应）。
        d、失败重传机制：客户端根据服务端响应包判断哪些包已经收到，未收到的数据包可以再次发送。
        e、有尝试次数限制，多次失败直接断开连接。
        f、动态调整等待ACK时间（TCP根据多次ACK响应时间动态调整等待时间）。
    3、快速恢复：同慢启动阶段，拥塞避免阶段操作一样。
        a、发生阻塞，阀值减小（减半），开始值以新阀值开始，以+1的方式增加。

IP网际协议位于网络层：IP寻址、路由、分包、组包等三个模块
    1、IP协议的作用是把各种数据包传送给对方（路由控制表）。
    2、网络中传输的网络包是根据MAC地址为传输方向的（以太网协议）。
    3、分片功能

使用ARP（用以解析地址的协议，根据通信方IP地址就可以反查出对应的MAC地址）协议凭借MAC地址进行通信。
    1、寻找IP对应设备的MAC地址（只支持IPV4）
    2、广播发送ARP包
    3、ARP缓存，定期清理缓存（几分钟）

ICMP协议：IP路由过程中异常失败，将回传ICMP包给发送端，确认失败原因
    1、IPv6环境没有ARP协议，采用ICMPV6协议替代实现MAC地址获取

NAT协议：地址转换协议（案例：IPv6到IPv4网络的消息通讯）
