HTTP（HyperText Transfer Protocol，超文本传输协议）是Web的规范，可以说Web是建立在HTTP协议上通信的。
1、HTTP协议版本：
    HTTP/1.0
    HTTP/1.1：
        HTTP/1.1开始默认支持持久连接（一次连接，传输多个资源请求），并支持管线化
        多个请求并发发送，服务端响应的话只能一个一个响应
    HTTP/2.0：
        2015年提出HTTP/2.0协议：使用stream流的形式传输，传输数据全部使用二进制（协议头和body）
        客户端可以多个请求同时发送，服务端响应也可以同步响应多个请求
        每个响应都有标识确认那些响应信息是同一批响应
2、HTTP是位于应用层的协议。
    a、底层是基于tcp/ip通讯层协议实现的
    b、与HTTP密不可分的协议：IP、TCP、DNS
        DNS：负责域名解析
            DNS服务位于应用层的协议。
            负责域名到IP地址间的解析服务（域名到IP的解析、IP到域名的解析）。
            DNS服务器是分层构建的，上层服务器记录下层服务器的地址。
            调用顺序由上层逐层往下调用，直到找到映射关系为止。
            DNS服务器采用UDP协议发送数据
3、HTTP协议用于客户端和服务端之间的通信（请求端为客户端、响应端为服务端）。
4、HTTP是不保存状态的协议（新的请求，会有新的响应，不会记录之前的状态）。
5、使用COOKIE进行状态管理。
6、资源访问：
    URI（统一资源标识符）：
        1、由某个协议方案表示的资源的定位标识符。
        2、URI用字符串标识某一互联网资源。
    URL（统一资源定位符）：
        1、URL表示资源的地点。
        2、URL是URI的子集。

HTTP请求：
请求格式：
    请求行：方法（GET/POST）、URI、http版本（HTTP/1.1）
    请求头：请求首部
    请求实体：body使用二进制数据
响应格式：
    响应行：http版本（HTTP/1.1）、响应状态码（200）、状态码原因短语（OK）
    响应头：响应首部
    响应实体：body使用二进制数据
请求类型：
    GET（请求服务器资源）：将查询字符添加到URL中，因此查询长度有限（1024字节）。
        从服务器获取数据
        通过URL提交数据，数据在URL中可见
        数据最多1024字节
        参数显示在地址栏，适用非敏感数据
    POST（传输实体主体）：请求的信息对客户端不可见，长度没有限制。
        向服务器传送数据
        数据放在FORM的数据体内提交
        数据量无限制
        适用敏感数据
    HEAD（获取报文首部，功能跟GET一样，只是不返回响应主体）
    OPTIONS（询问支持的方法、是否支持GET、POST等方法）
    DELETE（删除服务资源，与POST功能相反、没有验证机制存在安全问题）
    PUT（传输文件、没有验证机制存在安全问题）
    TRACE（路径追踪，可以追踪到客户端到服务器中间经过几次中转，容易引起跨域攻击）
    CONNECT（要求用隧道协议连接代理）

HTTPS协议：确保WEB安全（HTTP  +  加密  +  认证  +  完整性保护  =  HTTPS）
通信：HTTP协议——SSL协议——TCP协议——IP协议
实现：对称加密 + 非对称加密 + CA认证
过程：
    a、服务端发送客户端信息（数字证书签名信息 + 服务端公钥加密信息）
    b、客户端验证数字签名，获取服务端公钥
    c、使用公钥加密交换对称加密私钥信息
    d、使用对称私钥加密数据发送数据
